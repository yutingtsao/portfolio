About Me

UI/UX Designer 專業簡介

我所相信的設計

我是一位跨平台的 UI/UX 設計師，設計足跡橫跨 B2B 企業系統、B2C 行動應用，以及實體場域的嵌入式觸控介面（如加油、洗車等自助服務場景）。從螢幕後的複雜邏輯，到使用者指尖的每一次觸碰，我習慣在不同的使用情境之間穿梭，把混亂的需求梳理成清晰、可落地、且能長久延展的設計。

對我而言，設計從來不只是讓產品好看。它是一種翻譯——把商業目標翻譯成使用者願意停留的體驗，把複雜的功能翻譯成直覺的操作。我關注的不是單一畫面的精緻，而是整個流程能否讓人感覺「被理解」：少一點摩擦、多一點順暢，讓轉換與留存自然發生。

我的專案經理背景，讓我習慣在動手設計前先理解商業的全貌——資源、時程、各方角色的目標。這份視角讓我的設計決策不只服務美感，更能在真實的限制中落地生根。

我擅長的事

設計核心

我擅長運用色彩、排版與視覺層次，打造一致而具識別度的介面語言；並透過 Figma 建立從低保真 Wireframe 到高保真互動 Prototype 的完整流程。我重視設計系統的建構，因為它讓產品在擴張時依然保持秩序與一致——這對需要長期維護的企業系統與跨平台產品尤其關鍵。

系統與架構思維

面對複雜的功能，我會先回到資訊架構與使用者流程，繪製 User Journey 與 IA，找出真正的關鍵路徑。我相信好的結構是體驗的骨架，先把邏輯理順，視覺才有意義。

跨情境的設計判斷

不同平台有截然不同的設計約束。企業系統重視資訊密度與效率，行動應用講究情境與節奏，而實體場域的觸控介面則必須面對戶外光線、零學習成本、防呆與單一任務動線等挑戰。能在這些差異之間自由切換、做出對的取捨，是我最珍視的能力。

此外，我具備 HTML / CSS / JavaScript 與 Git 的基礎，能與前端工程師順暢協作，確保設計被精準地實現。

讓設計回應真實的工作流程

我相信好的設計來自對使用情境的理解。在一項企業管理平台的專案中，團隊持續收集企業客戶與第一線使用者的回饋——其中，客戶反映需要逐筆手動編輯車輛資料，在資料量龐大時格外耗時。我據此重新設計了批次編輯機制，讓使用者能一次處理多筆資料，顯著減少了重複性操作。這個調整提醒我：B2B 設計的價值，往往藏在「操作量級」這種容易被忽略的細節裡。

我如何工作

我的設計流程以「使用者需求 × 商業目標」為核心，從理解問題開始，逐步推進到可落地的解決方案：

  1. 需求與問題定義：釐清業務目標、使用者痛點與產品定位
  2. 流程與資訊架構：繪製 User Flow 與 IA，建立清晰的操作邏輯
  3. 視覺風格探索：定義色彩、字體與元件風格，建立一致的視覺語言
  4. 互動與動效設計：以細緻的反饋提升操作直覺度
  5. 低保真到高保真：Wireframe → Mockup → Prototype，逐步收斂
  6. 設計系統建立：建置元件庫與規範，提升一致性與重用率
  7. 開發協作與落地：與工程團隊緊密合作，確保設計精準實現
  8. 迭代優化：根據反饋與數據持續打磨，追求最佳體驗

我特別在意畫面的層次、動線的順暢，以及設計的延展彈性——讓作品不只在交付當下好看，更能隨產品一起成長。

設計之外的我

工作之外，我習慣讓自己慢下來——靜靜仰望星空、走進山林，或與我的三隻貓咪相處。我也喜歡冥想，以及閱讀與靈性相關的書籍，在這些安靜的片刻裡，重新校準自己的節奏與專注。

這些看似與設計無關的時光，其實深深影響著我的工作方式。冥想讓我更能在複雜的需求中保持清晰，閱讀則拓展了我理解人與情緒的角度。它們讓我更懂得傾聽——不只是使用者口中說出的需求，還有藏在每一段流程背後、未被言說的情緒與溫度。

我始終相信，能被記住的設計，往往來自那一點願意多想一步、多感受一分的同理心。
